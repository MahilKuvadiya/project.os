MRU PAGE REPLACEMENT ALGORITHM

The MRU (Most Recently Used) page replacement algorithm is a memory management technique used by operating systems to decide which pages to evict from memory when new pages need to be loaded.It replaces the page that has been most recently used, which means it keeps the pages that have been accessed frequently and recently in memory, and evicts the ones that have not been accessed recently.

  • Advantages

    • 1. It is useful for applications that require access to frequently used data because it keeps the most recently accessed data in memory. wh
    2. It improves the overall performance of the system.

  • Disadvantages

    • 1. It may not be suitable for applications that have a high rate of page faults because it can result in thrashing, where the system spends more time swapping pages in and out of memory than executing instructions.
    2. Average waiting time under the RR policy is often long.
