DISC SCHEDULING ALGORITHM

SCAN Disk scheduling Algorithm

This algorithm processes I/O requests in one direction along the disk, then reverses direction when it reaches the end of the disk. This algorithm ensures that all requests are eventually processed, but can lead to long wait times for requests at the end of the disk.

  • Advantages

    • 1.Implementation is easy.
    2. Requests do not have to wait in a queue.

  • Disadvantages

    • 1. The head keeps going on to the end even if there are no requests in that direction.

    C - SCAN Disk scheduling Algorithm

    This algorithm is similar to SCAN, but instead of reversing direction at the end of the disk, it jumps back to the beginning of the disk and processes requests in the same direction again. This algorithm ensures that all requests are processed in a timely manner.

  • Advantages

    • 1. The waiting time is uniformly distributed among the requests.
    2. Response time is good in it.

  • Disadvantages

    • 1. The time taken by the disk arm to locate a spot is increased here.
    2. The head keeps going to the end of the disk.
